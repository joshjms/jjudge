# Abstract

Online judge systems are essential infrastructure for competitive programming platforms, enabling automated evaluation of code submissions against predefined test cases. Existing solutions often rely on heavyweight virtualisation or privileged container runtimes, introducing operational complexity and security trade-offs. This project presents **JJudge**, a cloud-native, full-stack online judge system designed with security, scalability, and correctness as primary concerns.

The system is built on a **microservices architecture** comprising five loosely coupled, independently deployable components: a **Next.js 15** web frontend, a **Go** REST API server, an asynchronous **Go** worker service, a **gRPC**-based grader service, and **Lime** — a purpose-built, rootless container runtime implemented in C. Lime isolates code execution using Linux user namespaces, overlay filesystems, and cgroup v2 resource controls, achieving strong sandboxing without requiring elevated privileges. Resource limits including CPU time, wall-clock time, memory, and output size are enforced at the kernel level, enabling secure multi-tenant workload isolation.

The API server and worker communicate asynchronously via an **event-driven**, **message-queue** backbone — **RabbitMQ** in development and **GCP Pub/Sub** in production — decoupling submission ingestion from judging and enabling **horizontal scaling** of stateless worker instances. Test case files are managed in **cloud object storage** (**MinIO** or **GCS**), with an LRU cache on the worker tier reducing storage round-trips and improving throughput. Grading is handled by a dedicated **gRPC microservice**, enabling independent deployment, versioning, and scaling of the grading logic.

All services are **containerised** and orchestrated via **Docker Compose** for local development, with the architecture designed for seamless lift-and-shift to managed cloud infrastructure. The system supports both open practice and timed **contests**, with ICPC-style and IOI-style scoring, real-time leaderboards, and role-based access control. An end-to-end test suite validates the full **submission pipeline** from HTTP ingestion through verdict delivery.

This project demonstrates how a **cloud-native**, **event-driven microservices** architecture — combined with kernel-level isolation primitives — can deliver a secure, observable, and horizontally scalable code execution platform suitable for production deployment.
